2年生のまとめ

Apr 9, 2023 | 約1分で読めます

はじめに

ぼーっとしてたら 3 年生になって、授業も始まってしまいそうなので、2 年生の学生生活をまとめます。

移行

去年 3 月の頭に、悩んだ末最終的に、情報メディア創成学類を、移行第一志望とし、何とか希望も通り、情報メディア創成学類生として、2 年生を過ごすことになりました。

春学期授業

春学期の授業は、1 年次と同様に、オンライン授業がほとんどで、対面授業は実質体育だけ、という状況でした。いわゆる自分は「宅通」なので、非常にありがたい状況ではありました(サークル活動など、不便なこともたくさんありますが…) 1 年次と比べると、やはり専門的な授業が増えてきたなという印象でしたが、まだ自分の既に知っている範囲で何とか戦えていました。また、2 年次の授業で感じたこととして、初回授業でいきなり SSH に接続できるようにしないといけない、という授業など、CLI に授業で触れる機会が急に増えたように感じました(データ工学概論、プログラミングなどは代表例ですね) 一応 CLI を触るのは初めてではなかったので、そこまで大きな苦労はしませんでしたが、CLI などに全く触れたことのない人は、オンライン授業の中どうやって乗り越えたんだろうな…と少し気になっています。
また、やはりオンライン授業が中心になると、体育の授業とサークルといった対面活動が楽しかったです。こんな自分でも交流してくれる人がいるのだなと心が温かくなりました。 春学期に受けた必修授業を中心にまとめます。ネタバレ防止のため、授業の詳細やテスト内容については触れません。

線形代数 B

情報系のくせして数弱な自分は、この授業が一番春の中では苦労した気がします。問題の解法を考えること自体は楽しかったですが、それを TeX でレポートにまとめたり(手書きという手段もありますが、私の手書きでは脳内 OCR 不可能で落単の可能性がありました)、manaba を使わず直接 e-mail で転送する、独自の成績表示ページといった、少し面倒な点もありました(あと採点早くてびっくりしました…)

コンピューターシステムと OS

コンピューターに関する基礎知識や、アセンブリ言語を書かされました。また期末テストがあり、授業資料の確認も試験中にでき、比較的取り組みやすい講義だったかなと思います。あと試験会場が春日講堂で、「こういう広い講堂でテスト受けるの大学生感あるよなー」と思っていました。アセンブリ言語を書くのはパズルみたいで楽しかったです。

データ工学概論

1・2 限だったので朝早起きして同時双方向授業を受けるのが辛かった思い出があります… 各個人で、講義資料の演習に取り組んでいき、最終的に PHP と MySQL で自分オリジナルのシステムを作るという感じの授業でした。講義中は Slack で、匿名で質問ができ、それに TA さんや他の学生がお助けしていくという感じでした。こういうチャットでのわちゃわちゃは楽しかったです。 余談ですが自分は Slack では"John Doe"という名前で参加していました、私も他の方々のようにセンスのある面白い名前を考えていましたが、いいものが思いつきませんでした… 最終課題では、自分はチェスの棋譜を保存して、それぞれのムーブに注釈をつけることができる、というものを作りました。とりあえず手っ取り早く作れそうで、他とかぶらないだろうな、という感じで作りました(フロントエンドガチガチで作っていた人もいってびっくりしましたね…) テストは、テストで出されるであろう箇所が授業で何度も言及されていたので、取り組みやすかった気がします。

プログラミング

LaTex やったり、C 言語書いたりと、大変な授業でした。C 言語を初めてこの授業で書く、といった人は本当に大変だったんだろうな…と勝手に思っています。授業では、SSH につないで、Emacs で書くことが “想定” されていたと思いますが、普通に自分の好きな環境で書いちゃっていいと思います。自分は念のため、書いたコードは全学計算機上でもちゃんと動くことを確認してから提出していました。データ工学概論(この授業でも Emacs にふれることになっていました)といい、Emacs が好きな人が多いんですかね、Vim をずっと使っている自分としては少し悲しかったです。レポートのフィードバックが丁寧で有り難かったです(自分はコードの変数名に言及があったこともありました)

確率と統計

完全オンデマンドで、毎回の小テストに答えて、一部教科書の問題を自分で解く課題がある、という感じでした。小テストは一回あたりの問題が少ないので、一回やらかして小テストで 1 点をとったときは青ざめました。ちゃんと授業動画を見ていれば、落単はないと思います。

微分積分 B

正直授業動画をほとんど見ていなかったので、授業内容をほとんど覚えていません… 授業動画の公開の延期が何度もあったことだけは覚えています…

秋学期

夏休みや、自分のその他の活動は、秋学期の授業まとめの後に書きます。
秋学期になると、今までの対面とオンラインの割合をひっくり返したような履修になりました。対面授業だと、やはり大学生している感があって良いですね。でも私が授業で交流してたのは、学類生の中の、元総合生を中心とする極一部だったので、今後はいろいろな人のことを知れると良いなと思いました。秋学期は必修授業が少なかったので、印象に残っている授業をまとめたいと思います。

データ構造とアルゴリズム

講義と実習の 2 つに分かれていて、講義では、アルゴリズムについて学んでレポートにまとめる、実習ではテーマにしたがって、C でコードを書いていくという感じでした。実習は、講義資料が丁寧で、基本的に講義資料の指示に従って書いていけば、動くコードが書けた、という感じでした。
講義の方は、レポート・小テストが基本で、対面授業のグループワークも任意参加であったのですが、回を経るごとに人数が減っていき、自分も最終的に面倒くさくなって…という感じでした(本当に申し訳ありません)

情報数学 B

代数系などのお話が中心でした。やはり私は、数学の問題を考えるのは好きだけれど、それを TeX にまとめるのは嫌いなんだな…と感じました。曰く厳しい授業とのことですが、課題の問題を決められた数ちゃんとこなせば、単位はもらえる授業という印象でした。あとレポートのフィードバックが丁寧でした。有り難かったです(プログラミングの課題は言語自由だったので、Common Lisp で解いたら珍しいと言われました。Lisp 使う人あんまりいないのかな)

認知科学

期末テスト(対面持ち込みなし)で何を覚えればいいかわからなくて、何とか単位は来ましたが難しかったですね…正直自分は認知科学にあまり強い興味がないことがわかりましたね。

情報理論

期末テスト(対面持ち込みなし)とレポートがありました。テストに関しては、難しかったですが、事前に模擬試験でテストに似たような形式の問題を解く機会があったので良かったですね。授業資料がとてもわかりやすくてよかったです。レポート課題は使う言語自由だったので Lua で書きました。

情報社会と法制度

オンデマンドが基本で、一回だけ manaba の掲示板を使ったグループワークがありました。この授業で六法が必要とのことなので、2000 円ぐらいのデイリー六法を購入しましたが、結局授業で自分は使うことはありませんでした…今は本棚のインテリアになっています。

知的財産概論

オンデマンド授業・小テストのみによる評価でした。小テストは少し難しかったですが、回を経るごとになんとなく解き方がわかってきて、単位を取ることはできました。

情報デザイン I

各回ごとに、写真やプレゼンなどを提出してくという感じでした。毎回の授業で行われる先生の手直しが圧巻でした。自分は「言語学習について」というテーマでプレゼンを発表しました。実はアメリカかイギリスのヒップホップの聴き方とかについて語ろうと思っていましたが、面倒そうだったのでやめました。

その他の授業

春秋通して取り組んだ授業です。
